2) Limpe referências e redirecionamentos do “novo”

Rodar buscas rápidas:

rg -n "EstoqueSite|/estoque-v2" src
rg -n "Navigate\\s+to=.*(estoque|inventory-list-01)" src


Apague redirecionamentos que você adicionou especificamente para a migração.

Corrija qualquer <Link to="..."> que tenha sido alterado durante o teste (o certo é to="/estoque").

3) Valide visual e console (runtime)

Hard reload com cache desativado (DevTools > Network > Disable cache).

Console deve voltar a mostrar algo como:

at InventoryListPage1 (.../inventory-list-01/index.jsx:17:38)

Listings1 na árvore do React DevTools.

Layout volta ao “antigo”: mesma barra, grid, etc.

4) Remova o novo componente com cautela

Antes de excluir o arquivo, garanta que nada mais o importa:

rg -n "EstoqueSite" src


Se não houver referências:

Apague src/pages/estoque/EstoqueSite.jsx (e a pasta /estoque se existir só pra ele).

Remova CSS/JS auxiliares criados só para o novo (ex.: estoque-site.css|scss, helpers).
Busque por:

rg -n "estoque-site|page-hero|breadcrumb" src


Se preferir manter para testes futuros, renomeie para EstoqueSite.draft.jsx ou mova para src/_archive/EstoqueSite.jsx em vez de deletar.

5) (Opcional) Git/Versionamento

Faça um commit “Rollback: /estoque -> InventoryListPage1; remove EstoqueSite”.

Se tiver criado commits de migração, você também pode git revert <commit_da_mudança_da_rota>.
